# 🚀 xolve 公式説明書

最終更新: 2026-04-13  
対象: xolve デスクトップアプリ（Windows版）

---

## 1. このアプリでできること

xolve は、X（旧Twitter）運用を圧倒的に効率化し、戦略的なアカウント育成を支援するデスクトップアプリです。

- **投稿管理**: 単発投稿・連投（複数ポスト）の作成、**スレッド予約**（会話の親子連鎖）と **複数の単発を並べる**編集、カレンダーによる予約管理
- **AI アシスト**: AI による投稿のリライトや、AIシナリオ投稿（テーマから複数ポストを生成）
- **自動運用**: いいね・バズリライトをあなたの代わりに実行
- **マルチアカウント**: 複数アカウントを一括管理し、ログも一箇所で確認可能

---

## 2. はじめる前の準備

✅ **必要なもの**

- Windows PC
- 有効なライセンスキー
- X API 情報（アカウントごとの OAuth キーおよび Bearer Token）
- OpenAI API Key（AI 機能を利用する場合のみ必要）

---

## ⚠️ 初回起動時の注意

- **ライセンス認証**: 未認証の場合、最初に認証画面が表示されます。
- **データ保存**: 設定や投稿データはすべてローカル（あなたの PC 内）に安全に保存され、重要情報は暗号化されます。
- **保存形式（二層の SQLite）**: データは役割に応じて以下のファイルに分かれて保存されます。
- **共通設定 (`global.db`)**: アプリ全体の共通設定、自動運用の全体設定などが保存されます。
- **個別データ (`account-databases` 配下)**: 各 X アカウントの認証情報・予約・投稿ログなどは、アカウントごとに独立したファイル（`*.db`）で管理されます。

---

## 3. 初期設定（最短手順）

以下の 4 ステップで、すぐに運用を開始できます。

1. **ライセンス認証**: 発行されたキーを入力します。
2. **AI 設定**: `設定` 画面で OpenAI API Key を保存します。
3. **アカウント登録**: `X アカウント` 画面でトークン類を登録します。
4. **接続テスト**: 登録後、必ず「接続テスト」を実行して疎通を確認してください。

## 4. 画面ごとの使い方

### 📱 ダッシュボード

- 投稿文の作成、AI 候補生成、連投の編集（複数ポスト）
- **単発モード**: 複数ポストをそれぞれ別予約として並べ、ポスト間隔を指定可能
- **スレッドモード**: リプライ連鎖として投稿。日時は **ポスト1のみ**指定し、2通目以降はシステムが直前から約 **1〜30 秒後**に予約
- 予約投稿の作成（時刻・間隔・リピート設定）および即時投稿

### 👤 X アカウント

- アカウントの追加・管理
- **重要**: アイコン画像は入力された「ハンドル（@）」を元に取得します。別人の顔が出る場合は、ハンドル名が正しいか再確認してください。

### 📅 スケジュール

- 予約投稿を週/月単位で一覧表示
- 日時のドラッグ＆ドロップ編集や削除が可能（**右の「今後の投稿予定」**から週グリッドへドラッグして時刻変更。**右パネル上で離すと移動はキャンセル**）
- 予約の編集で **スレッド全体**（親子の連なり）をまとめて変更できる場合があります
- **CSV インポート**で日時・本文・メディアに加え、**スレッド／グループ列**でまとまりを指定できます

### 🤖 自動運用

- いいね・AI 等の ON/OFF 切り替え
- キーワード、実行間隔、1 日の上限値などをアカウントごとに細かく設定

### 📜 投稿ログ

- 成功・失敗の履歴確認
- エラー時はここから詳細なメッセージを確認できます。

## 8. よくある質問（FAQ）

### Q1. ライセンス認証に失敗する

- 入力ミス（前後の余計なスペース）がないか確認してください。
- ネットワーク状態を確認し、再試行しても失敗する場合はサポートまでご連絡ください。

### Q2. 投稿が実行されない / 失敗する

投稿がうまくいかない場合は、以下の 3 ステップを順に確認してください。

1. **設定の確認**: `X アカウント` 画面を開き、対象アカウントの OAuth（投稿用）および Bearer（検索用）が正しく登録されているか確認してください。
2. **接続テストの実行**: 同じ画面にある「接続テスト」ボタンを押し、正常に通信できているか（緑色のチェックが出るか）確認してください。
3. **エラーログの特定**: それでも失敗する場合は、`投稿ログ` 画面を開き、該当する行に表示されているエラーメッセージの内容を確認してください。

### Q3. 自動運用が動かない

- シミュレーション設定のままになっていないか確認してください。
- 対象アカウントが「有効」になっているか、また実行上限に達していないか確認してください。

### Q4. 投稿ログに 403 Forbidden が出る

多くの場合、X API 側の権限不足またはアプリ設定の不整合です。

- `X アカウント` 画面で設定を再確認してください。
- X Developer Portal 側で必要な権限（Read and Write）が有効か確認してください。

### Q5. アイコンが別人の顔写真になる

- アイコンは入力された「ハンドル（@）」を元に取得しています。OAuth や Bearer が正しくても、ハンドル名が別のユーザーだとその人の画像が表示されます。
- `X アカウント` 画面で、ハンドルが実際のアカウントと一致しているか確認してください。

### Q6. データはどこに保存されているか（バックアップしたいとき）

- **共通**: `global.db`（アプリのユーザーデータ領域内）。
- **アカウントごと**: 同じ領域の `account-databases` フォルダ内の `*.db`（1 アカウント 1 ファイル）。
- アカウントをアプリから削除すると、該当する **SQLite ファイルも削除**され、プロフィール画像のローカルキャッシュも消えます。手動で `.db` だけ消した場合は、起動時に **存在しない ID の参照が自動整理**される場合があります。

# <span id="first-step">🚀 運用開始後のファーストステップ（初日〜初週）</span>

最短手順の設定が終わったあとに実施することで、トラブルを防ぎ、安全に運用を開始するためのガイドです。

### 1. 日付・時刻の表示形式を合わせる（任意・推奨）
`設定` 画面で、お好みの表示形式を選択してください。ダッシュボードやスケジュール画面での誤読を防ぎ、運用ミスを減らすことができます。

### 2. 用途別 AI モデルの設定を確認する
`設定` の「AIモデル詳細」を開き、以下の各機能で適切なモデルが選択されているか確認してください。
- **手動リライト / シナリオ投稿**: 高品質な `gpt-4o` 推奨
- **バイラルリライト**: コストと速度重視の `gpt-4o-mini` 推奨
※設定変更後は必ず「AIモデル詳細を保存」を押してください。

### 3. まずは「シミュレーションモード」で様子を見る
本番の API を消費する前に、設定からシミュレーションを **ON** にして、ログの流れや動作イメージを掴んでください。詳細は [公式説明書 `README.md`] の「まずはシミュレーションモード」を参照してください。

### 4. テスト投稿を 1 本だけ実施する
大量の予約や自動運用を始める前に、まずは 1 本だけ「即時投稿」または「予約投稿」が成功することを確認してください。これで環境設定が完璧であることを証明してから本格運用に入るのが最も安全です。

## 🆘 よくあるエラーコードと対処法

画面に英語の数字（エラーコード）が出た場合は、以下を参考にしてください。

### ▼ 403 Forbidden（権限エラー）

- **意味**: X（Twitter）側の設定が「読み取り専用」になっています。
- **解決**: X Developer Portal で「Read and Write」権限を有効にし、連携をやり直してください。

### ▼ 401 Unauthorized（認証エラー）

- **意味**: 連携キーが古いか、間違っています。
- **解決**: `Accounts` 画面で連携情報を一度削除し、最新のキーを再設定してください。

### ▼ 429 Too Many Requests（回数制限）

- **意味**: 短時間に操作をやりすぎました（X 側の制限）。
- **解決**: 15 分〜24 時間ほど時間を置いてから再度お試しください。

### ▼ 402 / 403（OpenAI 支払い・制限エラー）

- **意味**: OpenAI 側の残高不足、またはプランによる利用制限です。
- **解決**: OpenAI のマイページでクレジット残高（Usage）を確認し、支払い設定を更新してください。

### ▼ MACHINE_MISMATCH（端末不一致）

- **意味**: 別の PC で同じライセンスが使われています。
- **解決**: 以前の PC で「ライセンス解除」を行うか、サポートまでお問い合わせください。

## 1. 起動時エラー・画面表示のトラブル

### 1-1. 起動直後にエラーが出る
* **症状:** 起動時に「例外が発生しました」等の表示が出る、またはログに `maintenance` 系のエラーがある。
* **原因:** 設定ファイル（JSON）の破損や読み込みエラー。
* **対処:**
    1. **完全終了:** タスクマネージャー（`Ctrl+Shift+Esc`）でアプリのプロセスを全て終了してから再起動。
    2. **設定の初期化:** 改善しない場合、設定ファイルを一度削除（または `.bak` から復元）して再起動してください。

### 1-2. 画面が真っ白 / 操作に応答しない
* **症状:** ウィンドウは開くが中身が白いまま、またはクリックしても反応がない。
* **原因:** 過去のプロセスが多重起動しているか、PCのメモリ不足。
* **対処:**
    1. **プロセスの終了:** タスクマネージャーでアプリ名を検索し、残っているプロセスを全て終了させてください。
    2. **PCの再起動:** 改善しない場合はPC自体を再起動してください。
    3. **セキュリティ確認:** セキュリティソフトがアプリをブロックしていないか確認してください。

## 2. ライセンス認証のトラブル

### 2-1. ライセンス認証に失敗する
* **原因:** 入力ミス、通信遮断、または別PCでの使用。
* **対処:**
    * **空白の確認:** キーを貼り付ける際、**前後に不要なスペース**が入っていないか厳密に確認してください。
    * **通信環境:** インターネットに接続されているか確認してください。
    * **エラーコード:** 「MACHINE_MISMATCH」と出る場合は、以前の端末で解除が必要です。

### 2-2. ライセンス解除ができない
* **症状:** 「解放機能が登録されていません」等のエラーが出る。
* **対処:**
    * **最新版への更新:** 古いバージョンを使っている可能性があります。最新のビルドに更新し、アプリを再起動してから再度お試しください。

---

## 3. X（Twitter）投稿・自動運用エラー

### 3-1. 投稿が失敗する（APIエラー）
* **対処:** `Accounts` 画面で「連携テスト」が通るか確認してください。
* **エラーコード早見:**
    * **401:** 認証切れ。アカウント設定を一度削除し、**再連携**してください。
    * **403:** 権限不足。X Portalで **「Read and Write」** 権限があるか確認してください。
    * **429:** 回数制限。15分〜24時間ほど時間を置いてから再試行してください。

### 3-2. 自動運用が動かない
* **仕様:** 自動運用は **「アプリが起動している間」** のみ動作します。
* **対処:**
    1. **スリープ設定:** 運用中はPCがスリープしないよう設定してください。
    2. **アプリモード:** ブラウザ単体ではなく、必ずビルドされたアプリ本体から実行してください。

## 4. AI（OpenAI）連携のトラブル

### 4-1. AI機能が動かない
* **原因:** キーの不備、またはOpenAI側の支払い状況。
* **対処:**
    1. **キーの確認:** OpenAIのAPIキーが正しく設定されているか（空白がないか）確認してください。
    2. **残高確認:** OpenAIマイページで、クレジット残高が不足していないか確認してください。

---

## 5. CSV・予約管理のトラブル

### 5-1. CSV読み込みエラー
* **原因:** フォーマット（列順やヘッダー名）の不一致。
* **対処:**
    * 必ず配布されている **サンプルCSV** の形式（列の並び順・ヘッダー名）を守ってください。
    * 保存形式は **「CSV UTF-8 (コンマ区切り)」** を選択してください。

### 5-2. 予約カレンダーの挙動
* **症状:** ドラッグ操作で意図せず時刻が変わった。
* **対処:** 最新版では右パネルに戻すとキャンセルされます。意図せず確定した場合は、予約一覧から手動で修正してください。

---

## 6. データベース (SQLite) の不整合

### 6-1. アカウント一覧が表示されない
* **原因:** 手動でのDBファイル移動や、ファイル破損。
* **対処:**
    1. **自動修復:** 最新版へアップデートし、再起動して自動マイグレーションを待ちます。
    2. **再登録:** 改善しない場合、対象アカウントを一度削除し、アプリ内から再登録してください。
