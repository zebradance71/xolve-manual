# 🚀 xolve 公式説明書

最終更新: 2026-03-28  
対象: xolve デスクトップアプリ（Windows版）

---

## 1. このアプリでできること

xolve は、X（旧Twitter）運用を圧倒的に効率化し、戦略的なアカウント育成を支援するデスクトップアプリです。

- **投稿管理**: 単発投稿・スレッド（連投）の作成、カレンダーによる予約管理
- **AI アシスト**: AI による投稿のリライトや、バズるスレッドの自動生成
- **自動運用**: いいね・フォロー・リプライ・バズリライトをあなたの代わりに実行
- **マルチアカウント**: 複数アカウントを一括管理し、ログも一箇所で確認可能

---

## 2. はじめる前の準備

✅ **必要なもの**

- Windows PC
- 有効なライセンスキー
- X API 情報（アカウントごとの OAuth キーおよび Bearer Token）
- OpenAI API Key（AI 機能を利用する場合のみ必要）

---

## ⚠️ 初回起動時の注意

- **ライセンス認証**: 未認証の場合、最初に認証画面が表示されます。
- **データ保存**: 設定や投稿データはすべてローカル（あなたの PC 内）に安全に保存され、重要情報は暗号化されます。

---

## 3. 初期設定（最短手順）

以下の 5 ステップで、すぐに運用を開始できます。

1. **ライセンス認証**: 発行されたキーを入力します。
2. **AI 設定**: `設定` 画面で OpenAI API Key を保存します。
3. **アカウント登録**: `X アカウント` 画面でトークン類を登録します。
4. **接続テスト**: 登録後、必ず「接続テスト」を実行して疎通を確認してください。
5. **テスト投稿**: `ダッシュボード` で投稿を作成し、動作を確認します。

---

## 🛡️ 重要：まずは「シミュレーションモード」で確認

アカウント凍結リスクを下げるため、新しい設定を試す際は必ずシミュレーションから開始してください。

- 実際の投稿・アクションは行われません。処理の流れだけを安全に確認できます。
- キーワード設定や実行間隔のミスを本番前に発見できます。

**推奨フロー**

1. シミュレーションを **ON** にする
2. 1 日分の動作を確認し、`投稿ログ` をチェック
3. 問題がなければ本番へ切り替える

---

## 4. 画面ごとの使い方

### 📱 ダッシュボード

- 投稿文の作成、AI 候補生成、スレッド編集
- 予約投稿の作成（時刻・間隔・リピート設定）および即時投稿

### 👤 X アカウント

- アカウントの追加・管理
- **重要**: アイコン画像は入力された「ハンドル（@）」を元に取得します。別人の顔が出る場合は、ハンドル名が正しいか再確認してください。

### 📅 スケジュール

- 予約投稿を週/月単位で一覧表示
- 日時のドラッグ＆ドロップ編集や削除が可能

### 🤖 自動運用

- いいね・フォロー・AI リプライ等の ON/OFF 切り替え
- キーワード、実行間隔、1 日の上限値などをアカウントごとに細かく設定

### 📜 投稿ログ

- 成功・失敗の履歴確認
- エラー時はここから詳細なメッセージを確認できます。

---

## 5. 日常運用のおすすめ

### 🛡️ 安全な運用フロー

1. **シミュレーション**: まずは ON で挙動を確認
2. **本番切替**: 問題がなければ実運用を開始
3. **ログ確認**: `投稿ログ` を毎日チェック
4. **トラブル対応**: エラーが出た場合は解決ガイド（トラブルシュートガイド）を参照

### ⚠️ 凍結リスクを下げるポイント

- **投稿間隔**: 人間らしい自然な間隔を設定してください。
- **日次上限**: アプリ側の上限値を守って運用してください。
- **急激な増加の回避**: 投稿量を一気に増やさず、徐々に拡大するのがコツです。

## 8. よくある質問（FAQ）

### Q1. ライセンス認証に失敗する

- 入力ミス（前後の余計なスペース）がないか確認してください。
- ネットワーク状態を確認し、再試行しても失敗する場合はサポートまでご連絡ください。

### Q2. 投稿が実行されない / 失敗する

投稿がうまくいかない場合は、以下の 3 ステップを順に確認してください。

1. **設定の確認**: `X アカウント` 画面を開き、対象アカウントの OAuth（投稿用）および Bearer（検索用）が正しく登録されているか確認してください。
2. **接続テストの実行**: 同じ画面にある「接続テスト」ボタンを押し、正常に通信できているか（緑色のチェックが出るか）確認してください。
3. **エラーログの特定**: それでも失敗する場合は、`投稿ログ` 画面を開き、該当する行に表示されているエラーメッセージの内容を確認してください。

### Q3. 自動運用が動かない

- シミュレーション設定のままになっていないか確認してください。
- 対象アカウントが「有効」になっているか、また実行上限に達していないか確認してください。

### Q4. 投稿ログに 403 Forbidden が出る

多くの場合、X API 側の権限不足またはアプリ設定の不整合です。

- `X アカウント` 画面で設定を再確認してください。
- X Developer Portal 側で必要な権限（Read and Write）が有効か確認してください。

### Q5. アイコンが別人の顔写真になる

- アイコンは入力された「ハンドル（@）」を元に取得しています。OAuth や Bearer が正しくても、ハンドル名が別のユーザーだとその人の画像が表示されます。
- `X アカウント` 画面で、ハンドルが実際のアカウントと一致しているか確認してください。
