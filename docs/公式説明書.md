# xolve 公式説明書

最終更新: 2026-03-28  
対象: xolve デスクトップアプリ（Windows）

---

## 1. このアプリでできること

xolve は、X（旧Twitter）運用をまとめて管理するデスクトップアプリです。

- 投稿の作成（単発・スレッド）
- 予約投稿（カレンダー管理）
- 複数アカウント運用
- AIによるリライト/スレッド生成
- 自動運用（いいね・フォロー・リプライ・バズリライト）
- 投稿ログ確認

---

## 2. はじめる前の準備

## 必要なもの

- Windows PC
- 有効なライセンスキー
- X API 情報（アカウントごと）
- OpenAI API Key（AI機能を使う場合）

## 初回起動時の注意

- ライセンス未認証の場合、先にライセンス入力画面が表示されます。
- 設定や投稿データはローカルDBに保存されます。
- APIキー類は暗号化して保存されます。

---

## 3. 初期設定（最短手順）

以下の順で設定するとスムーズです。

1. ライセンスを認証する
2. `設定` で OpenAI API Key を保存する（AIを使う場合）
3. `X アカウント` で OAuth/Bearer を登録する
4. 接続テストを実行して疎通確認する
5. `ダッシュボード` で投稿作成し、予約または今すぐ投稿する

---

## 重要: まずはシミュレーションモードで確認

xolve では、実投稿を行わずに挙動確認できる **シミュレーションモード** を使えます。  
アカウント凍結リスクを下げるため、最初は必ずシミュレーションで確認してください。

- 実際の投稿・アクションを行わずに、処理の流れを確認できる
- キーワード設定、実行間隔、上限値、対象アカウントの設定ミスを本番前に見つけやすい
- 自動運用を本番化する前の安全確認として推奨

推奨手順:
1. シミュレーションを ON
2. 1 日分の想定で動作確認
3. 投稿ログと実行結果を確認
4. 問題がなければ本番へ切替

---

## 4. 画面ごとの使い方

## ダッシュボード

- 投稿文の作成、スレッド編集、テンプレート適用
- AI候補生成、スレッド生成
- 予約投稿の作成（時刻・間隔・リピート設定）
- 今すぐ投稿

## X アカウント

- アカウント追加・編集・削除
- 投稿用 OAuth 情報の登録
- 検索用 Bearer Token の登録
- 接続テスト
- 自動運用向けの設定（必要項目）

## 投稿テンプレート

- 定型文テンプレートの作成・管理
- 変数入りテンプレートの活用
- CSVインポートで一括登録（予約作成）

## スケジュール

- 予約投稿の一覧表示（週/月）
- 予約内容の編集
- 日時変更
- 不要な予約の削除

## 自動運用

- いいね / フォロー / AIリプライ / バズリライトのON/OFF
- キーワード、間隔、上限、フィルタ設定
- シミュレーション実行
- 検索キューの確認・手動処理

## 投稿ログ

- 成功/失敗の確認
- エラー内容の確認
- シミュレーションログと本番ログの確認

## 設定

- OpenAI API Key
- テーマ/表示形式
- AI関連オプション
- 保守・メンテナンス操作

---

## 5. 日常運用のおすすめ

## 安全な運用フロー

1. まずシミュレーションで挙動確認
2. 問題なければ本番運用へ切替
3. 投稿ログを毎日確認
4. エラーが出たらトラブルシュートを参照

## 凍結リスクを下げるポイント

- 投稿間隔を詰めすぎない
- 日次上限を守る
- 検索・投稿の急激な増加を避ける
- 本番運用前に設定漏れ（OAuth/Bearer）を確認する

---

## 6. アップデート通知について

- 起動時に最新版チェックを行います。
- 新バージョンがある場合、画面上部に通知が表示されます。
- 通知のリンクから配布ページを開けます。

---

## 7. エラー監視（運用者向け）

- 予期しないエラーは匿名化したうえで監視基盤へ送信される場合があります。
- 送信対象は運用に必要な最小情報のみです（エラー種別、メッセージ、OS、アプリ版など）。
- APIキーやトークン等の秘匿情報は送信対象に含めない設計です。

## 8. よくある質問（FAQ）

## Q1. ライセンス認証に失敗する

- 入力ミス（前後スペース）を確認
- ネットワーク状態を確認
- 再試行しても失敗する場合はサポートに連絡

## Q2. 投稿できない

- `X アカウント` の OAuth/Bearer 登録を確認
- 接続テストを実行
- `投稿ログ` のエラーメッセージを確認

## Q3. 自動運用が動かない

- シミュレーション設定のままになっていないか確認
- 対象アカウントが有効か確認
- 上限に達していないか確認

## Q4. 投稿ログに `403 Forbidden` が出る

- 多くの場合、X API 側の権限不足またはアプリ設定不整合です。
- まず `X アカウント` 画面で OAuth/Bearer の設定を再確認してください。
- 次に、X Developer Portal 側で必要な権限（投稿/読み取り）が有効か確認してください。
- 変更後は接続テストを再実行し、改善しない場合はトークン再発行を検討してください。

## Q5. アイコンが別の人の顔写真になる

- 一覧や切り替えに表示するアイコンは、**そのアカウント行に入力した「ハンドル（@）」** を X API で引いて取得しています。**OAuth や Bearer のトークンが正しくても、ハンドルが別のユーザーだと、その人のプロフィール画像が表示されます。**
- **「アカウント」画面で、ハンドルが実際に投稿しているユーザーと同じ @ か** を確認してください。修正後は自動で取り直されます（古いキャッシュはハンドルと紐づけて無効化されます）。
