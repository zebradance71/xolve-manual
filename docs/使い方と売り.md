# xolve 使い方と売りのポイント

> この文書は **現在の実装（ソース）** に合わせて更新しています。詳細な数式・端の挙動は [スケジュール挙動まとめ](./SCHEDULE_BEHAVIOR.md) および [自動運用のアカウント別上限](./AUTO_ENGAGEMENT_PER_ACCOUNT.md) を参照してください。

## 概要

**xolve** は、X（旧 Twitter）の投稿・予約・自動運用をまとめて扱う **Windows 向け Electron デスクトップアプリ**です。OpenAI によるリライト・スレッド生成・自動リプライ／バズリライト、凍結対策、複数アカウント、ローカル DB（SQLite / Prisma）によるデータ保持が特徴です。

### 画面構成（サイドバー）

| パス | 名称 | 主な用途 |
|------|------|----------|
| ダッシュボード | 投稿ワークスペース | 下書き・AI・予約・今すぐ投稿 |
| X アカウント | アカウント一覧・詳細 | OAuth / Bearer / プロキシ / AI 用 `rewritePrompt` |
| 投稿テンプレート | テンプレート＋CSV | 定型文の管理、CSV 一括予約 |
| スケジュール | カレンダー | 予約の確認・日時変更・編集 |
| 自動運用 | キーワード自動実行 | いいね・フォロー・リプライ・バズリライト・検索キュー |
| 投稿ログ | ログ一覧 | 手動・自動・今すぐ投稿の結果 |
| 設定 | 全体設定 | OpenAI・表示・検索オプションなど |

## 使い方（機能ごと）

### 1. ダッシュボード（メインの投稿ワークスペース）

| 操作 | 説明 |
|------|------|
| **アカウント切替** | 左上のスイッチャーで投稿に使うアカウントを選択（予約・今すぐ投稿に使用） |
| **複数ポスト** | ポストを追加し、スレッド風に編集。各ポスト **140 文字以内**（コード上の上限） |
| **ポスト間隔** | 2 本目以降は「直前のポストから N 分後」で指定。プリセット（15 分〜4 時間）と一括リセットあり（詳細は [SCHEDULE_BEHAVIOR.md](./SCHEDULE_BEHAVIOR.md)） |
| **画像添付 UI** | ポストごとに jpg / png / gif を最大 4 枚まで（ドロップ・プレビュー）。※現状の「スケジュールに追加」フローでは `mediaJson` が空で送られるため、**予約実行での画像投稿は本文中心の利用を前提**とする（UI とサーバ連携の差に注意） |
| **テンプレートから作成** | モーダルでテンプレート選択・変数置換 |
| **AI候補を生成** | 入力全文を対象に、OpenAI で複数バリエーション（アカウントの `rewritePrompt` があれば反映） |
| **スレッド生成** | テーマと本数を指定し AI が本文を生成（間隔は現在のデフォルト間隔プリセットに追随） |
| **外部から本文を流し込み** | 自動運用の検索キューなどから「ダッシュボードへ遷移して下書きに載せる」導線あり（ルータ state 経由） |
| **凍結対策** | スケジューラーで有効時、ポスト1の基準時刻に **±5〜10 分**（デフォルト）のランダムオフセット。有効／無効・範囲はパネルで変更可能 |
| **リピート予約** | 毎日／平日／週末／毎週／カスタムなどで複数日分を一括生成（重複スキップ等。詳細はスケジュールドキュメント） |
| **今すぐ投稿** | 即時投稿（**本文のみ**。成功時は X で開くリンク） |
| **スケジュールに追加** | 基準日時・凍結対策・リピート設定を反映してカレンダーへ登録。自動運用が **シミュレーション ON** のときはシミュ用予約として記録 |

### 2. X アカウント

- 複数アカウントの登録・削除（上限あり・UI で案内）
- **投稿用**: OAuth 1.0a（Consumer Key/Secret・Access Token）
- **検索用**: Bearer Token（recent search 等）
- **X API v2 アプリ**: Client ID / Secret（接続テスト等で利用）
- プロキシ、User-Agent、Cookie、**AI リライト用システムプロンプト**（`rewritePrompt`）
- 従来 OAuth と **X API 専用の疎通テスト**（`xapi:testConnection`）
- アカウント単位の **凍結回避パラメータ**（最小間隔・1 日あたりの投稿ソフト上限・ジッター幅など。スキーマ上のフィールド）

### 3. 投稿テンプレート

- タブ **「テンプレート」**: 名前・カテゴリ・本文（`{{name}}` 等の変数・スピンテキスト）
- タブ **「CSV」**: CSV をドロップして検証し、予約一括登録（本文・任意の日時・メディア1〜4列）。順次間隔はテンプレート画面の設定に準拠。凍結対策オプションあり

### 4. スケジュール

- 週／月表示で予約イベントを表示
- ドラッグやモーダルで日時・本文・凍結対策を編集
- 複数件削除など

### 5. 自動運用

| 項目 | 説明 |
|------|------|
| **アクション** | いいね / フォロー / AI リプライ / **バズリライト**（検索ヒットを AI リライトして自アカウントで投稿）を個別に ON/OFF |
| **ターゲットキーワード** | 複数行。実行時にその中からランダム選択して検索 |
| **フィルタ** | 最小フォロワー、除外ワード、リプライ対象の最小いいね／リポスト（0 で無制限） |
| **絵文字** | AI リプライの絵文字レベル（なし／控えめ／適度） |
| **実行間隔** | 分単位のランダムレンジ（例: 3〜12 分） |
| **日次上限** | いいね・フォロー・リプライ・バズリライトの 1 日あたり上限（共通設定） |
| **アカウント別上限** | 自動運用画面から、いいね／フォロー／リプライをアカウントごとに上書き（空欄＝共通値）。バズリライト上限は共通設定＋アカウント側の上書き（DB）を集計に反映 |
| **実行アカウント** | 本番時はチェックしたアカウントのみ。OAuth で投稿系、Bearer で検索をローテーション |
| **シミュレーション** | ON で実 API を叩かず動作・ログ・統計の流れを確認 |
| **タイマー** | ON で定期実行。検索は別スケジュール（下記）と連動 |
| **検索スケジュール** | `off` / 節約 1 回（20 時）/ 標準 2 回 / トレンド 3 回 / カスタム時刻 |
| **検索取得件数** | X API `max_results`（1〜100）。リプライ・バズリライト＋下限フィルタ時は多め取得に振る舞い調整あり |
| **即時検索** | キューを空にしてから検索し、ヒットで置換 |
| **検索ヒットキュー** | 一覧・使用済み表示、行の削除、手動バズリライト投稿、リライトして **バズ下書き** に保存、**ダッシュボードへ送る** |
| **本日カウントリセット** | アプリ内の本日カウンタのみリセット（X 上の履歴とは無関係） |
| **フリーズガード表示** | 有効アクションと上限に対する消化率を可視化（複数アカウント集計） |

検索が長期失敗した場合、バズリライト自動投稿を一時停止するなど、キューと検索の整合を取るロジックがメインプロセス側にあります。

### 6. 投稿ログ

- 成功／失敗、エラー内容、ポスト ID
- **シミュレーション由来**と本番を分けて参照

### 7. 設定

| カテゴリ | 内容 |
|----------|------|
| **API設定** | **OpenAI API Key のみ**（保存は暗号化）。X API は設定画面ではなく **各アカウント詳細**で入力 |
| **全般** | 日付表示形式、テーマ（システム／ライト／ダーク）、AI リライト最大文字数、AI ログ保存の ON/OFF、**X 検索の段階フォールバック**（既定 OFF。ON でヒット不足時に追加の検索 API 呼び出し＝課金増）、**グローバル NG ワード**（カンマ／改行区切り）、暗号化の説明 |
| **プロキシ** | 文言どおり **アカウント画面で設定** |
| **ライセンス** | プレースホルダ（今後追加予定の UI メッセージ） |

### 8. ワーカー・通知

- **ワーカー状態**の購読（IPC プッシュ）
- **ワーカーログ**（待機・投稿開始・完了・エラー）
- X API の **402/429** 相当時はアプリ側でイベント通知（残高・レート制限の案内）

## 売りのポイント

### 1. AI で投稿〜エンゲージまで一気通貫

- **ダッシュボード**: リライト候補・スレッド生成
- **自動運用**: AI リプライ、検索キューからの **バズリライト**（オリジナル投稿として再構成）
- アカウントごとの **カスタム `rewritePrompt`** で口調を揃えられる

### 2. 凍結対策の多層化

- 予約の **ランダムバッファ**（既定 ±5〜10 分、変更可）
- 実行側の **追加ジッター**（スケジュールドキュメント参照）
- 自動運用の **ランダム間隔・日次上限・フリーズガード表示**

### 3. 検索と運用の設計

- **検索スケジュール**と **アクション実行**を分離しつつ同一キーワード基盤
- **即時検索**・キューの手動操作・バズ下書き・ダッシュボードへの転送で、自動と手動のハイブリッド運用がしやすい
- **段階フォールバック**は任意（課金とトレードオフを UI で明示）

### 4. 複数アカウント

- 投稿・検索・自動運用のローテーション
- **アカウント別の自動運用 1 日上限**（いいね／フォロー／リプライは画面から編集しやすい構成）

### 5. ローカルファースト

- Electron + SQLite（開発時 `prisma/dev.db`、配布後はユーザーデータ領域）
- API キーは **暗号化して DB 保存**（`ENCRYPTION_KEY` は `.env` 側で管理）

### 6. 体験

- ガラス調 UI・ダークモード
- トースト・ワーカーログで進行状況が追いやすい

## 推奨ワークフロー例

1. **設定**で OpenAI API Key を保存
2. **X アカウント**で OAuth・Bearer・必要ならプロキシを入力し、疎通確認
3. **投稿テンプレート**（任意）→ **ダッシュボード**で作成 → 凍結対策・時刻・リピートを確認して **予約**
4. **自動運用**でキーワード・アクション・上限・検索スケジュールを設定し、シミュで確認のうえ本番へ
5. キューに溜まったヒットを **手動バズリライト** または **ダッシュボードへ送って** 編集・予約
