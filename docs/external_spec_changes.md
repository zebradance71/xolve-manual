# External Spec Changes Log

目的: 外部仕様変更（X API、スクレイピング対象、外部サービス、IPC連携条件など）による障害を即時記録し、暫定回避と恒久対応を管理する。

## 記録ルール

- 外部仕様変更が原因の fail を検知したら当日中に記録する。
- 最低限、以下を埋める:
  - 発生日/検知日
  - 影響範囲
  - 何が変わったか
  - 暫定回避策
  - 恒久対応方針

## エントリーテンプレート

```md
## [YYYY-MM-DD] <短いタイトル>

- 検知日時:
- 種別: X API / Scraping / External Service / IPC / Other
- 影響範囲:
  - 例) 投稿機能、AutoEngagement、設定保存、E2E
- 症状:
  - 例) 401/403/429、タイムアウト、スキーマ不一致
- 何が変わったか:
  - 例) エンドポイント応答仕様、認証要件、DOM構造、レート制限
- 暫定回避策:
  - 例) リトライ間隔拡大、対象機能の一時停止、フォールバック経路
- 恒久対応方針:
  - 例) パーサ更新、APIバージョン移行、監視閾値見直し
- 関連ログ:
  - `logs/api-activity.log`:
  - 代表レスポンス/エラー:
- 検証:
  - 実行コマンド:
  - 結果: pass / fail
- 対応状況: open / monitoring / fixed / closed
```
