# スケジュール挙動まとめ

ポスト（ダッシュボード）と CSV インポートのスケジュール挙動を、それぞれ分けてまとめます。

---

## 1. ポスト（ダッシュボード）の挙動

### 基準となる時刻

- **ポスト1の投稿時刻**: ユーザーがスケジューラーパネルで指定した日時
- ポスト2以降は「直前のポストから何時間何分後」で相対的に設定

### ポスト間隔の設定

| 項目 | 説明 |
|------|------|
| 表示 | ポスト2: 「ポスト1から 1時間30分後」、ポスト3: 「ポスト2から 2時間後」… のように、直前ポストからの間隔で表示 |
| 編集 | 各ポストのバッジをクリックして、時間・分を入力。直前ポストからの間隔として保存 |
| プリセット | 15分、30分、1時間、1時間30分、2時間、2時間30分、3時間、3時間30分、4時間 |
| 一括リセット | 2ポスト以上あるとき、「ポスト間隔」で選択した間隔ですべてリセット（0, 間隔, 2×間隔…） |
| 新規ポスト追加 | 直前2ポストの間隔を引き継ぐ。1ポストのみのときはデフォルト間隔（1時間） |

### スレッド作成・テンプレート等（間隔の適用元）

以下はいずれも「ポスト間隔」で選択したデフォルト間隔（0, 間隔, 2×間隔…）で `delayMinutes` が設定される。

| 操作 | 説明 |
|------|------|
| スレッド生成（AI） | ThreadGeneratorModal で生成したポストに、選択中の間隔を適用 |
| テンプレート適用 | テンプレートから作成モーダルで選択したとき、分割されたポストに適用 |
| AI候補の選択 | AIリライトの候補を選択したとき、分割されたポストに適用 |
| 他ページからの追加 | viral 等の他ページからダッシュボードに遷移してポストが追加されるとき、`nextDelayMinutes` でデフォルト間隔を使用 |

### 凍結対策（ランダムバッファ）

- ポスト1の基準時刻に ±min〜max 分のランダムオフセットを加算
- ポスト2以降は、そのずれた時刻を基準に `delayMinutes` 分後に実行
- デフォルト: ±5〜10分。有効/無効、範囲はスケジューラーパネルで設定

### 追加の揺らぎ（ジッター）

- 各ポストの実行時刻に ±2分、±30秒のジッターが内部で加算される（凍結対策とは別）

### リピート

- 毎日/平日/週末/毎週/カスタムで、1ヶ月分を一括生成
- 各日程の基準時刻に凍結対策を適用
- 重複時間帯はスキップ

## 2. CSV インポートの挙動

### 行ごとの扱い

| 日時欄 | 挙動 |
|--------|------|
| 指定あり | その日時を基準として投稿。凍結対策を適用した `effectiveAt` で実行 |
| 空欄 | 順次投稿モード。基準時刻 + 順次間隔で自動補完 |

### 順次投稿モードの基準時刻

1. **日時指定ありの行が1行以上ある場合**
   - 全行のうち最も遅い指定時刻を `maxFixed` とする
   - 基準 = `maxFixed + 順次間隔`
   - 1行目の空欄: 基準 + 0
   - 2行目の空欄: 基準 + 1×順次間隔
   - 3行目の空欄: 基準 + 2×順次間隔 …

2. **すべての行が空欄の場合**
   - 基準 = インポート実行時の現在時刻
   - 1行目の空欄: 基準（≈今）
   - 2行目: 基準 + 1×順次間隔
   - 3行目: 基準 + 2×順次間隔 …

### 順次間隔の設定

- プリセット: 15分、30分、1時間、1時間30分、2時間、2時間30分、3時間、3時間30分、4時間（ポストと同じ）
- デフォルト: 1時間
- CSV 読み込み後の設定パネルで選択

### 例（順次間隔を1時間にした場合）

| 行 | 日時欄 | 投稿予定 |
|----|--------|----------|
| 1 | `2025-03-23T10:00` | 10:00（凍結対策適用） |
| 2 | （空欄） | 11:00（10:00 + 1時間） |
| 3 | （空欄） | 12:00 |
| 4 | （空欄） | 13:00 |

| 行 | 日時欄 | 投稿予定 |
|----|--------|----------|
| 1 | （空欄） | インポート実行時 |
| 2 | （空欄） | 実行時 + 1時間 |
| 3 | （空欄） | 実行時 + 2時間 |

### 凍結対策

- ポストと同じ仕組み。各基準時刻に ±min〜max 分のランダムを加えて `effectiveAt` を決定

## 3. 共通事項

- **凍結対策**: ダッシュボード・CSV・カレンダー編集で同じ `FreezeProtectionConfig` を利用
- **間隔プリセット**: `DELAY_INTERVAL_PRESETS`（`post.ts`）で共通定義
- **実装**: `scheduleHandlers.ts`（`createFromEditor`, `importFromCsv`）、`scheduling.ts`（`applyRandomBuffer`）
